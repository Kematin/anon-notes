# 1. Общая информация о проекте

- Название: Сервис анонимных записок
- Основная цель: Создание сервиса для отправки анонимных записок с возможностью самоуничтожения
- Платформа: Веб

# 2. Основные функциональные требования

## 2.1 Функциональность записок

1. Получение зашифрованной записки `GET /notes/{id}`:
	- Получение созданной анонимной записки в зашифрованном виде
	- ID передается в зашифрованном виде
2. Создание записки `POST /notes`:
	 - Создание новой анонимной записки
	 - Возвращает зашифрованное ID
	 - Возможность задать время саморазрушения
	 - В теле запроса передается зашифрованный текст
3. Удаление записки `DELETE /notes/{id}`:
	- Возможность сразу удалить записку
	- Возможность удалить записку через определенный интервал
4. Расшифровка происходит на стороне браузера (клиента)
5. Возможные временные интервалы для саморазрушения:
	- Сразу
	- Минута
	- Час
	- День
	- Неделя

## 2.2 Функциональность шифрования

1. Шифрование записок
	 - Требуется хранить содержимое записок в зашифрованном виде
	 - Должно быть реализовано сквозное шифрование через секретную фразу + AES-256
2. Шифрование id
	- id возвращается в зашифрованном виде (UUID)
	- id встраивается в ссылку для получения нужной записки
3. Полный цикл жизни записки:
	-  Создание:
		1. Текст + Пароль AES-256 шифрование на стороне клиента
		2. Шифрованный текст -> `POST /notes`
		3. `POST /notes` -> `encrypted_id`
	- Чтение:
		1. `GET /notes/{id}` -> шифрованный текст с сервера
		2. Ввод пароля + AES-256 расшифровка в браузере
		3. Полученный текст на стороне клиента
## 2.3 Функциональность самоуничтожения

1. Записки само уничтожаются через определенной интервал
2. Сторона клиента (Frontend) несет ответственность за создание задачи по удалению, с помощью отправки запроса на путь `DELETE /notes/{id}`
